# Changelog

All notable changes to this project are documented here. The format is based on
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project
adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

## [0.7.0] - 2026-06-27

### Added
- **Interactive rolls.** Rolling a Move or casting a spell now opens a modifier
  dialog where the player can add Forward / Relationship and toggle their
  applicable passives — heritage topic forwards (e.g. +1 on Muggle topics, -2
  for Pureblood) and the House "+1 Forward to <attribute>" benefit (only shown
  for the matching attribute and if that benefit was chosen). Shift-click a Move
  to skip the dialog (fast roll).
- Spell **Spell-Bank / Signature** bonus and the **year penalty** are applied
  automatically and itemized in the roll breakdown.
- Roll result cards now show a **breakdown** of every modifier and a **Reroll**
  button (covers the Slytherin reroll benefit).
- The character data model computes `system.rollMods` (the passives offered in
  the dialog) from heritage and the chosen House benefit.

### Fixed
- **Creation wizard would not open** ("Template part body must render a single
  HTML element"). The wizard template is now wrapped in a single root element, as
  ApplicationV2 parts require.

### Added
- **Half-Blood heritage choice**: the heritage card (and wizard) now let
  Half-Blood characters pick one bonus and one drawback from the Muggle-Born /
  Pureblood lists via dropdowns; the other heritages remain fixed (shown
  automatically). Stored in `system.details.heritageBonus` / `heritageDrawback`.

## [0.6.1] - 2026-06-27

### Fixed (rulebook compliance for character creation)
- Attributes are now capped at **+2 / -2 during creation** (the +4 cap only
  applies later via advancement), in the wizard inputs and on apply.
- Spell step now reflects the rulebook count: **3** first-year spells, **+1 for
  Pureblood** (and a note that Half-Blood gets +1 if they took the Pureblood
  bonus spell).

### Added (completing the manual's creation steps)
- **Background step** (rulebook Step 7) in the wizard: parents/siblings/you,
  written to the character's background.
- Pet **personality** and **look** fields in the wizard pet step.
- Heritage **bonus/drawback** shown in the wizard's attributes step.

## [0.6.0] - 2026-06-27

### Added
- **Guided character-creation wizard** (`module/apps/creation-wizard.mjs`): a
  9-step ApplicationV2 launched from the character sheet (wand button). Steps:
  identity & heritage, attributes, House & benefit, traits, appearance, wand,
  pet, spells, summary. It is non-destructive — nothing is written to the actor
  until *Finish*. Automations:
  - Applies the heritage's base attribute spread and rolls starting galleons.
  - Derives the **wand wood from the birthday** via the Celtic calendar; rolls
    core and length.
  - Picks traits and first-year spells **from the compendiums** (so they arrive
    translated and iconed), creating the owned items on finish.
  - Creates the wand and pet items and fills House, benefit, looks and birthday.
- `config.wandWoods` / `wandCores` / `wandFlexibilities` / `months` data.

### Fixed
- Description/biography/background editors no longer render empty when editing.
  They are now built with `HTMLProseMirrorElement.create()` (helper
  `module/helpers/editor.mjs`) instead of a bare `<prose-mirror value>` tag,
  which failed to load existing content.

### Added
- **House benefit automation**: the benefit field is now a dropdown populated
  from the character's selected House (`config.houseBenefits`, localized).
- **Heritage (guided creation) automation**: a Linaje card shows the heritage's
  base attribute spread, starting-galleons formula, and bonus/drawback. An
  *Apply heritage* action sets the base attribute spread and rolls starting
  galleons (with confirmation); a *Roll galleons* action re-rolls them.
  Backed by `config.heritageData`.

### Added
- **Spanish translation of all compendium content** via optional [Babele](https://foundryvtt.com/packages/babele)
  integration — when Babele is installed, content shows in the client's language
  automatically (UI was already localized). Covers all 176 spell descriptions,
  the 13 moves, 72 traits, 16 potions and the 4 House journal pages.
  - Registered through `babele.init` in `module/waw.mjs`
    (`setSystemTranslationsDir("lang/babele")`).
  - Translation files generated by the build pipeline from
    `tools/data/i18n-es.mjs` into `lang/babele/es/witchcraft-and-wizardry.<pack>.json`.
  - Babele declared as a *recommended* module relationship in `system.json`.

## [0.3.0] - 2026-06-27

### Added
- Four new compendiums, generated from the rulebook via a generalized build
  pipeline (`tools/build-packs.mjs`):
  - **Moves** — the 13 basic moves as rollable `move` items (attribute + 10+/7-9/6- outcomes).
  - **Personality Traits** — 36 positive + 36 negative `trait` items for character creation.
  - **Potions** — a curated set of 16 canon Wizarding World potions.
  - **The Four Houses** — a JournalEntry pack (ghost, common room, entrance, benefits).
- All packs declared in `system.json` and grouped under one compendium folder.
- Per-pack build scripts (`pack:spells`, `pack:moves`, …) plus `build:packs`.

### Changed
- `tools/build-spells.mjs` replaced by the generic `tools/build-packs.mjs`,
  which builds every content pack from `tools/data/*.mjs`.

## [0.2.0] - 2026-06-27

### Changed
- Redesigned the character sheet: elegant, house-themed dark UI with a five-tab
  layout (Stats / Magic / Character / Inventory / Story), attribute cards that
  list their moves, condition pills, an experience pip track, spell-bank rows
  with signature/bank toggles, and an inline relationships editor.

### Fixed
- Localized actor/item subtype names (`TYPES.Actor.*` / `TYPES.Item.*`) so the
  sheet titles and the "Create Actor/Item" dialogs no longer show raw keys.

### Added
- `potion` item subtype (effect / quantity / price).
- Character fields from the official sheet: house benefit, birthdate, favorite
  class, electives, appearance, and a separate long-form background.

## [0.1.0] - 2026-06-27

### Added
- Initial system scaffold for Foundry VTT v13/v14.
- Manifest (`system.json`) declaring `character` / `npc` actors and
  `spell` / `trait` / `wand` / `pet` / `equipment` / `move` items.
- ESM entry point with `CONFIG` registration and Handlebars helpers.
- Data models (`TypeDataModel`) for every actor and item subtype.
- ApplicationV2 sheets for the character, NPC and items.
- PbtA 2d6 roll engine for Moves and Cast a Spell.
- **Complete spell compendium**: all 176 spells (years 1–7) compiled to a
  LevelDB pack (`packs/spells`) and declared in the manifest. Authored from a
  single source of truth (`tools/data/spells.mjs`) via `tools/build-spells.mjs`.
- **Icon mapping**: built-in Foundry icons assigned per document subtype
  (`config.defaultIcons`) and per spell classification (`config.spellTypeIcons`),
  applied automatically on document creation through `_preCreate`.
- Build scripts: `build:spells`, `pack`, `unpack`, `build`.
- English and Spanish localization.
- Documentation: `README.md`, `docs/DATA_MODEL.md`, `packs/README.md`.
